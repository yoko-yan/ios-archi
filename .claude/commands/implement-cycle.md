## 入力: $ARGUMENTS

## サイクル

1. **実装** → 2. **ビルド確認** → 3. **テスト** → 4. **レビュー** → 5. **動作確認** → 6. **完了**
（問題あれば修正して繰り返し）

## 各フェーズ

### 1. 実装
コード作成・編集

### 2. ビルド確認
`build-errors` サブエージェントを実行。エラーがあれば修正。

### 3. テスト
`run-tests` サブエージェントを実行。失敗があれば修正。

### 4. レビュー
`self-review` サブエージェントを実行。問題があればPhase 2に戻る。

### 5. 動作確認（UI変更時のみ）
`verify-app` サブエージェントを実行してスクリーンショット確認。

### 6. 完了報告
変更ファイル、品質確認結果を報告。

## エラー時
- 3回試行しても解消しない場合はユーザーに相談
